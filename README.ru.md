# Средство для пересохранения ресурсов Serious Engine 2+

[![.NET Version](https://img.shields.io/badge/dotnet-8.0-purple)](https://dotnet.microsoft.com/)

**[English](README.md)** | **[Русский](README.ru.md)**

Десктопное WPF-приложения для пересохранения файлов ресурсов движков Serious Engine версий 2 и выше под новым именем с обновлением ссылок на них в других файлах.

## Функционал

- **Групповое переименование:** Быстрое переименование файлов путем замены подстрок в исходном имени.
- **Поиск и обновление ссылок:** Автоматический поиск файлов, ссылающихся на пересохраняемые файлы с последующим обновлением ссылок.
- **Пересохранение:** Сохранение ресурсов под новым именем с обновлением внутренних ссылок на другие.

## Начало работы

### Требования

- Windows 10/11
- Для запуска портативной версии требуется установленный [.NET 8.0 Runtime](https://dotnet.microsoft.com/ru-ru/download/dotnet/8.0/runtime)

### Установка и запуск

1.  **Скачайте сборку:**  
    [Скачать последнюю версию](https://github.com/0ther1/SeResResaver/releases/latest)  
    Или перейдите в раздел [Releases](https://github.com/0ther1/SeResResaver/releases)

    Запустите **SeResResaver.exe**.

## Использование
      
1. **Выбор файлов для пересохранения**
    a. Укажите каталог игры, файлы которой вы собираетесь пересохранять.
    b. Добавьте файлы, которые вы хотите пересохранить (по одному или папками).
    c. Переименуйте добавленные файлы (вручную или используя множественное переименование).
    d. Опционально отметьте файлы, которые нужно удалить после успешного пересохранения.
2. **Выбор файлов для обновления ссылок**
   Добавьте файлы, в которых вы хотите обновить ссылки (по одному или папками).
   *Примечание:* в списке появляются только те файлы, в которых были найдены ссылки на пересохраняемые ресурсы.  
   *Важно:* Пересохраняемые файлы обновляют ссылки друг на друга автоматически.
3. **Пересохранение**
    a. Выберите из списка игру, для которой вы пересохраняете файлы.
    b. Нажмите кнопку **Начать**.

После завершения процесса пересохранения будет показан результат - **Завершено** или **Завершено с ошибками**. При наличии ошибок они будут отображены в таблице.
